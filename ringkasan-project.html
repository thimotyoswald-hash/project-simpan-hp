1. NAMA PROJECT

SIMPEN HP

Sistem Informasi Manajemen Peminjaman HP

2. DESKRIPSI PROJECT

SIMPEN HP adalah aplikasi web yang dirancang untuk mengelola peminjaman HP siswa di sekolah. Sistem ini memudahkan guru piket untuk mencatat siswa yang meminjam HP mereka, memantau durasi peminjaman, mencatat pengembalian, dan menganalisis pola peminjaman siswa melalui laporan yang komprehensif.

Karakteristik Utama:

  • Tanpa sistem login - akses langsung ke dashboard
  • HP yang dipinjam adalah milik siswa sendiri
  • Tracking real-time durasi peminjaman
  • Laporan dan analisis perilaku peminjaman
  • Design dark mode yang clean dan modern

3. FITUR / HALAMAN DI PROJECT

A. Dashboard (index-dark.html)

Metric cards: HP dipinjam, peminjaman hari ini, total siswa, HP lama • Quick access menu ke 3 fitur utama • Navigation sidebar dengan 7 menu

B. Peminjaman HP (peminjaman-dark.html)

Form input: Pilih siswa, jam pinjam, alasan peminjaman, catatan • Auto-fill kelas berdasarkan siswa • Validasi form dan success notification • Generate nomor referensi otomatis (LOAN-YYYYMMDD-###)

C. Daftar HP Dipinjam (daftar-dipinjam-dark.html)

Tabel real-time list HP yang sedang dipinjam (12 data sample) • Filter by: siswa, kelas, status • Search functionality • Durasi auto-calculate dan update setiap menit • Status highlight: normal, warning (>1 jam), danger (>2 jam) • Button kembalikan untuk setiap data

D. Pengembalian HP (pengembalian-dark.html)

Dropdown search peminjaman aktif • Detail card auto-load dari peminjaman terpilih • Input kondisi HP: Baik, Sedikit Rusak, Rusak Parah, Hilang • Conditional field detail kerusakan • Auto-hitung durasi peminjaman • Summary pengembalian sebelum submit

E. Laporan Harian (laporan-harian-dark.html)

Filter by tanggal • Stat cards: Total peminjaman, HP dikembalikan, alasan terbanyak, durasi rata-rata • Tabel detail peminjaman harian (6 data sample) • Export PDF & Excel buttons

F. Laporan Siswa (laporan-siswa-dark.html)

Filter: Kelas, Periode, Sort by • Tabel analisis perilaku per siswa (6 data sample) • Highlight row untuk siswa dengan frekuensi tinggi • Detail button per siswa • Export functionality

G. Manajemen Data Siswa (data-siswa-dark.html)

Button tambah siswa baru • Filter: Search nama/NIS, filter kelas, filter status • Import CSV button • Tabel CRUD siswa (8 data sample) • Status badge: Aktif (hijau), Nonaktif (kuning) • Pagination (1-8 dari 1,240 siswa)

5. STRUKTUR DATABASE

Table: students

Column Type Constraint Keterangan
id INT PRIMARY KEY Auto increment
nis VARCHAR UNIQUE NOT NULL Nomor Induk Siswa
nama VARCHAR NOT NULL Nama lengkap siswa
kelas VARCHAR NOT NULL XI-A, X-B, XII-C
no_hp VARCHAR NULL Nomor HP siswa
status ENUM DEFAULT 'aktif' aktif / nonaktif
created DATETIME AUTO Waktu data dibuat
updated DATETIME AUTO Waktu data diupdate

Table: device_loans

Column Type Constraint Keterangan
id INT PRIMARY KEY Auto increment
student_id INT FK/NOT NULL Referensi students
tanggal_pinjam DATE NOT NULL Tanggal pinjam
jam_pinjam TIME NOT NULL Jam mulai pinjam
jam_kembali TIME NULL Jam pengembalian
alasan VARCHAR NOT NULL Tugas/Emergency/Main
kondisi_akhir VARCHAR NULL Kondisi dikembali
status ENUM NOT NULL aktif / selesai

Relationships

students (1) ──── (∞) device_loans

Satu siswa bisa memiliki banyak record peminjaman HP

Setiap peminjaman pasti ada siswa yang terkait

Query Contoh untuk Laporan

1. Laporan Harian

SELECT s.nis, s.nama, s.kelas, dl.jam_pinjam, dl.jam_kembali,
       dl.alasan, dl.kondisi_akhir,
       TIMEDIFF(dl.jam_kembali, dl.jam_pinjam) as durasi
FROM device_loans dl
JOIN students s ON dl.student_id = s.id
WHERE DATE(dl.tanggal_pinjam) = CURDATE()
ORDER BY dl.tanggal_pinjam DESC;

2. Statistik Siswa

SELECT s.nis, s.nama, s.kelas,
       COUNT(dl.id) as total_peminjaman,
       MAX(dl.jam_pinjam) as jam_puncak
FROM students s
LEFT JOIN device_loans dl ON s.id = dl.student_id
WHERE s.status = 'aktif'
GROUP BY s.id
ORDER BY COUNT(dl.id) DESC;

3. Peminjaman Aktif

SELECT s.nis, s.nama, s.kelas, dl.jam_pinjam,
       dl.alasan, TIMEDIFF(NOW(), dl.jam_pinjam) as durasi
FROM device_loans dl
JOIN students s ON dl.student_id = s.id
WHERE dl.status = 'aktif'
ORDER BY dl.jam_pinjam ASC;
